2. 주문 생성 (Place Order)

sequenceDiagram
  participant FE as Frontend
  participant OC as OrderController
  participant SVC as OrderService
  participant INV as InventoryPort→InventoryAdapter
  participant CAT as ProductQueryPort→CatalogProductAdapter
  participant REPO as OrderRepository→OrderPersistenceAdapter
  participant EVT as OrderEventPublisher

  FE->>OC: POST /api/orders
  OC->>SVC: place(PlaceOrderCommand)
  SVC->>INV: reserve(items)
  INV-->>SVC: 임시 홀드
  SVC->>CAT: findProduct/findOption
  CAT-->>SVC: 상품/옵션(단가)
  SVC->>SVC: Order 생성 (PENDING_PAYMENT)
  SVC->>REPO: save(order)
  SVC->>INV: createReservationForOrder(hold, orderId)
  SVC->>EVT: publishOrderPlaced()
  EVT-->>SVC: (Modulith outbox에 event_publication 기록)
  SVC-->>OC: OrderResponse
  OC-->>FE: 201 Created
  Note over SVC,INV: 도중 예외 시 InventoryPort.release(hold)로 롤백

3. 결제 승인 (Toss confirm)

sequenceDiagram
  participant FE as Frontend (Toss 위젯)
  participant OC as OrderController
  participant SVC as OrderService
  participant PAY as PaymentGatewayPort→TossPaymentAdapter
  participant TOSS as Toss Payments API
  participant INV as InventoryPort
  participant EVT as OrderEventPublisher

  FE->>OC: POST /api/orders/{id}/confirm-payment (paymentKey, amount)
  OC->>SVC: confirm(orderId, customerId, paymentKey, amount)
  SVC->>SVC: 소유자/상태(PENDING_PAYMENT)/금액 검증
  SVC->>PAY: confirm(paymentKey, orderId, amount)
  PAY->>TOSS: POST /v1/payments/confirm
  TOSS-->>PAY: 200 OK
  PAY-->>SVC: 승인 완료
  SVC->>SVC: order.markPaid(paymentKey) → PAID
  SVC->>INV: confirmByOrderId() (예약 재고 확정 소진)
  SVC->>EVT: publishOrderPaid()
  SVC-->>OC: OrderResponse
  OC-->>FE: 200 OK

4. 주문취소 + 환불 (PAID → CANCELED)

sequenceDiagram
  participant FE as Frontend / 관리자
  participant OC as OrderController / OrderAdminController
  participant SVC as OrderService
  participant PAY as PaymentGatewayPort→TossPaymentAdapter
  participant TOSS as Toss Payments API
  participant INV as InventoryPort
  participant EVT as OrderEventPublisher

  FE->>OC: POST .../cancel
  OC->>SVC: cancel() / cancelByAdmin()
  SVC->>SVC: 상태 PAID 검증 (아니면 OrderCancelNotAllowedException)
  SVC->>PAY: cancel(paymentKey, reason)
  PAY->>TOSS: POST /v1/payments/{paymentKey}/cancel
  alt 환불 성공 또는 ALREADY_CANCELED_PAYMENT
    TOSS-->>PAY: 200 / 이미취소(성공 매핑)
    PAY-->>SVC: 성공
    SVC->>INV: restockByOrderId() (확정재고 재입고)
    SVC->>SVC: order.markCanceled() → CANCELED
    SVC->>EVT: publishOrderCanceled()
    SVC-->>OC: OrderResponse
  else 그 외 PG 실패
    TOSS-->>PAY: 실패
    PAY-->>SVC: PaymentRefundFailedException
    SVC-->>OC: 취소 중단 (로컬 상태 변경 없음)
  end
  Note over SVC: 재입고/저장 단계 실패 시 환불은 이미 완료된 상태 —
동일 취소 API 재호출로 수렴(Toss가 ALREADY_CANCELED_PAYMENT 반환)
